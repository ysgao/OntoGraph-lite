# Implementation Plan: Unsaved Entity Editor Changes Warning

**Branch**: `022-unsaved-changes-warning` | **Date**: 2026-07-01 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/022-unsaved-changes-warning/spec.md`

## Summary

Guard all entity navigation paths in the Entity Editor against silent data loss. When the user navigates away from an entity that has unsaved edits, a modal dialog (Save / Discard / Cancel) intercepts the navigation. The implementation reuses the existing webview dirty-state computation (`checkForChanges()`) and the existing save message pipeline (`handleSave()` → `'save'` message) via three new lightweight IPC messages (`queryDirty` / `dirtyState` / `requestSave`).

## Implementation Design

### A. New IPC Messages (`EntityEditorMessages.ts`)

Add three message interfaces (see `contracts/webview-ipc.md` for full contract):

```typescript
// Extension → Webview
interface QueryDirtyMessage { type: 'queryDirty' }
interface RequestSaveMessage { type: 'requestSave' }

// Webview → Extension
interface DirtyStateMessage { type: 'dirtyState'; isDirty: boolean }
```

Register `QueryDirtyMessage` and `RequestSaveMessage` in the `ExtensionToWebviewMessage` union.
Register `DirtyStateMessage` in the `WebviewToExtensionMessage` union.

### B. Webview Handler (`EntityEditorApp.ts`)

In the `window.addEventListener('message', ...)` handler, add two new cases:

```typescript
case 'queryDirty': {
  const isDirty = JSON.stringify(getCurrentState()) !== lastSavedStateString;
  vscode.postMessage({ type: 'dirtyState', isDirty });
  break;
}
case 'requestSave':
  handleSave();
  break;
```

**Bug fix bundled with this feature**: `renderEntity()` currently does not set
`lastSavedStateString`, so the editor is immediately dirty after any entity loads.
Add `lastSavedStateString = JSON.stringify(getCurrentState())` at the end of
`renderEntity()` to start the editor in a clean state.

### C. Extension Guard Function (`EntityEditorPanel.ts`)

Add module-level state:

```typescript
let pendingNavigationIri: string | null = null;
let dirtyQueryResolve: ((isDirty: boolean) => void) | null = null;
```

Add `queryDirty()` — sends message, returns `Promise<boolean>`:

```typescript
async function queryDirty(panel: vscode.WebviewPanel): Promise<boolean> {
  return new Promise(resolve => {
    dirtyQueryResolve = resolve;
    panel.webview.postMessage({ type: 'queryDirty' });
  });
}
```

Handle `'dirtyState'` in the existing `handleMessage()` switch:

```typescript
case 'dirtyState':
  if (dirtyQueryResolve) {
    const resolve = dirtyQueryResolve;
    dirtyQueryResolve = null;
    resolve(msg.isDirty);
  }
  break;
```

Export `guardedShowEntityInfo()`:

```typescript
export async function guardedShowEntityInfo(
  context: vscode.ExtensionContext,
  model: OntologyModel,
  iri: string,
  cancelRevealCallback?: () => void,
): Promise<void> {
  const panel = getPanel();
  const currentIri = getLastIri();

  if (!panel || !currentIri || iri === currentIri) {
    showEntityInfo(context, model, iri);
    return;
  }

  const isDirty = await queryDirty(panel);
  if (!isDirty) {
    showEntityInfo(context, model, iri);
    return;
  }

  const label = getEntityLabel(model, currentIri) ?? currentIri;
  const choice = await vscode.window.showWarningMessage(
    `"${label}" has unsaved changes. What would you like to do?`,
    { modal: true },
    'Save',
    'Discard',
  );

  if (choice === 'Save') {
    pendingNavigationIri = iri;
    panel.webview.postMessage({ type: 'requestSave' });
    // Navigation completes in the augmented 'save' case of handleMessage()
  } else if (choice === 'Discard') {
    showEntityInfo(context, model, iri);
  } else {
    // Cancel / dismissed
    cancelRevealCallback?.();
  }
}
```

Augment the `'save'` case in `handleMessage()` — after the existing save success path:

```typescript
// After sendLoadEntity(p, ...) refresh call:
if (pendingNavigationIri) {
  const targetIri = pendingNavigationIri;
  pendingNavigationIri = null;
  showEntityInfo(context, model, targetIri);
}
```

On write failure, clear `pendingNavigationIri` and show an error notification without navigating.

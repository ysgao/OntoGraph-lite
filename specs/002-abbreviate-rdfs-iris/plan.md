## Summary

Currently only `rdfs:label` is written as an abbreviated token in OWL documents; `rdfs:comment`, `rdfs:seeAlso`, and `rdfs:isDefinedBy` are written as full bracketed IRIs (`<http://www.w3.org/2000/01/rdf-schema#comment>`). This feature extends the abbreviation rule to all four RDFS built-in annotation property IRIs, both on the write path (`abbreviateIri` / `iri()`) and on the read path (annotation-line parsers that recognise abbreviated tokens). The fix is applied in three files: `AnnotationSync.ts`, `AxiomSync.ts`, and `FunctionalSerializer.ts`. See [research.md](./research.md) for root-cause analysis and approach selection.

All 8 must be confirmed failing before Phase 2 begins.

### Phase 2 — Green: Implementation

Three-file change, same pattern in each:

1. **`src/sync/AnnotationSync.ts`**:
   - Remove `const RDFS_LABEL = '...'`
   - Add `RDFS_PREFIX`, `RDFS_ANN_TO_TOKEN`, `RDFS_TOKEN_TO_IRI` module-level constants
   - Update `abbreviateIri`: `RDFS_ANN_TO_TOKEN.get(iri) ?? \`<${iri}>\``
   - Line ~120 (`parseFunctionalAnnotationItem`): `RDFS_TOKEN_TO_IRI.get(propToken) ?? resolveIri(propToken, prefixes)`
   - Line ~257 (`parseManchesterAnnotationLine`): same pattern
   - Update remaining `RDFS_LABEL` references (in `entityAnnotationPairs`) to use `RDFS_ANN_TO_TOKEN` keys or the full prefix constant

2. **`src/sync/AxiomSync.ts`**:
   - Remove `const RDFS_LABEL = '...'`
   - Add same three constants
   - Update `abbreviateIri`
   - Update `entityAnnotationSegs` where it references `RDFS_LABEL`

3. **`src/serializer/FunctionalSerializer.ts`**:
   - Remove `const RDFS_LABEL = '...'`
   - Add same three constants
   - Update `iri()` function
   - Update `generateEntityCluster` where it references `RDFS_LABEL` directly

# Implementation Plan: Remove Authoring-UI Integration — Standalone Extension

**Branch**: `015-remove-authoring-ui-integration` | **Date**: 2026-06-10 | **Spec**: [spec.md](spec.md)

## Summary

Remove the two remaining integration artifacts that were introduced during OntoGraphEditor (`ontograph.ontograph-editor-extension`) development. The majority of integration code was already removed from the standalone; only `updateGraphPanel()` (exported but never used) and a vestigial comment remain. See [research.md](research.md) for root-cause analysis.

The "command already exists" activation error occurs because OntoGraphEditor re-registers all `ontograph.*` commands — a collision that cannot be resolved from OntoGraph-lite alone without renaming command IDs. The two extensions are mutually exclusive; OntoGraphEditor already handles this via `--disable-extension=ysgao.ontograph-lite` in its launch config.

## Implementation Phases

### Phase 1 — Remove `updateGraphPanel` export

`updateGraphPanel()` is exported but never imported anywhere in the standalone codebase. It was added for `OntoGraphEditor/extension/src/graph/activateGraph.ts` to call when the model changes. Remove the function entirely.

**Before** (`openVisualization.ts:69-77`):
```typescript
/** Update the graph panel when the model changes (called from extension.ts) */
export function updateGraphPanel(
  model: OntologyModel,
  focusIri?: string,
  preferredLang = 'en',
): void {
  if (!panel) { return; }
  sendGraph(panel, model, focusIri, 2, { showInferred: true, showDisjoint: false }, preferredLang);
}
```

**After**: deleted.

**Verification**: `npm run compile` — no imports of `updateGraphPanel` exist, so no cascading errors.

### Phase 2 — Clean vestigial comment in `nodeClicked` handler

The `nodeClicked` branch in the graph webview message handler contains a leftover comment from when the `ontographEditor.ipcRoute` dispatch was removed. The branch itself is fine (handles a valid message type), but the comment is misleading.

**Before** (`openVisualization.ts:57-58`):
```typescript
} else if (msg.type === 'nodeClicked') {
  // Nothing for now — could reveal in tree
}
```

**After**:
```typescript
} else if (msg.type === 'nodeClicked') {
  // intentional no-op
```

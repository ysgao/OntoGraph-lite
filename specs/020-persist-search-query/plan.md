# Implementation Plan: Persist Entity Search Query

**Branch**: `020-persist-search-query` | **Date**: 2026-06-30 | **Spec**: [spec.md](spec.md)

**Input**: Feature specification from `specs/020-persist-search-query/spec.md`

## Summary

Add in-session persistence of the last entity search string to the `ontograph.searchEntity` QuickPick command. When the QuickPick is reopened, it pre-populates with the last query and immediately displays matching results. The stored query resets to empty when a new ontology is loaded.

## Technical Context

**Language/Version**: TypeScript 5 (strict mode)

**Primary Dependencies**: VS Code Extension API (`vscode.QuickPick`), Vitest (testing)

**Storage**: Module-level `string` variable in `src/extension.ts` — in-session only, no disk persistence

**Testing**: Vitest unit tests in `src/`

**Target Platform**: VS Code Extension Host (Node.js)

**Project Type**: VS Code extension

**Performance Goals**: QuickPick opens with pre-filled query and initial results within the same latency as a fresh search (<500 ms)

**Constraints**: In-session only (no `workspaceState` / `globalState`); query reset on ontology change

**Scale/Scope**: Single string variable; negligible memory footprint

## Implementation Notes

### Change 1 — `src/extension.ts`: add storage variable

Add near the top of `activate()` (or at module level alongside other globals):

```
let lastSearchQuery = '';
```

Reset it in the `setRefreshAllViews` callback (already called on every successful ontology load/reload):

```
lastSearchQuery = '';
```

### Change 2 — `src/extension.ts`: pre-populate QuickPick

Inside `ontograph.searchEntity` command handler, after `createQuickPick()` and before `show()`:

```
quickPick.value = lastSearchQuery;
if (lastSearchQuery) {
  // manually trigger search with stored query — onDidChangeValue is not fired on programmatic set
  runSearch(lastSearchQuery);
}
```

### Change 3 — `src/extension.ts`: update storage on input change

In the existing `onDidChangeValue` handler:

```
lastSearchQuery = query;   // add this line alongside existing debounce/search call
```
